# AutoLock - Codebase Review & TODO List

## 1. Errors & Bugs

- `core/main.py`: `_hotkey_launch_client` hardcodes League of Legends client paths to `C:\Riot Games\Riot Client\RiotClientServices.exe`, `D:\`, `E:\`. This is error-prone if the client is installed elsewhere. It should query the registry or standard locations more robustly, or rely solely on user configuration.
- `services/api_handler.py`: `psutil` operations inside `LCUClient.connect` can throw `psutil.AccessDenied` when querying process cmdlines. The fallback is good, but it logs an error level sometimes when it should be a warning.
- `ui/components/experimental_profile_list.py`: The time difference function `_format_time_ago` is called but might not exist in the same module (it relies on an implicit import or is missing in the provided snippet - needs verification).
- `services/stats_scraper.py`: `StatsScraper._try_lolalytics` and `_try_metasrc` use fragile regex/HTML parsing to extract win rates. Website structure changes will break this. Consider using an official API or a more robust parser like BeautifulSoup if an API isn't available.

## 2. Redundancies & Tech Debt

- `ui/theme/` vs `ui/tokens/`: Theming seems to be split or transitioning between standard hardcoded colors and design tokens (`factory.py` uses `TOKENS`). There's some redundancy in how UI components fetch colors.
- `utils/logger.py`: Provides both `Logger` (a class with static methods) and a module-level `log` function. Pick one logging convention and stick to it to reduce confusion.
- `services/asset_manager.py`: Extensive duplicated logic for downloading images and parsing file paths. Could abstract a generic `_download_and_cache_image` method instead of repeating the pattern for champion square, splash, and loading art.
- `services/automation.py`: `_tick()` performs multiple HTTP requests to LCU sequentially using `executor.submit`. While they are in a thread pool, they could be gathered/awaited more efficiently if the architecture allowed asyncio.

## 3. Performance Optimizations

- `services/asset_manager.py`: Caching mechanism uses `Image.Resampling.BICUBIC` for resizing and does alpha manipulation in python. While LUT is used for opacity, applying it to every image loaded can slow down initial UI rendering. Pre-processing and caching the final styled images to disk (in `cache/` dir) would be significantly faster on subsequent loads.
- `services/api_handler.py`: Process scanning `psutil.process_iter(["name"])` is called frequently when disconnected. This is an expensive operation on Windows. Consider reducing the polling frequency when the client is definitively not running, or using a less expensive check first.
- `core/main.py`: `docking_loop` polls `FindWindowW` and `GetWindowRect` in a tight loop with a 0.5s sleep. This is relatively cheap, but could be optimized by only updating geometry if the target window actually moved.


## 4. UI / UX Improvements

- **Compact Mode**: The compact mode transition (`core/main.py`) shrinks the window and removes window borders (`overrideredirect(True)`). Dragging is supported via `_setup_window_dragging`, but returning to full mode can be unintuitive if the hotkey isn't remembered. Ensure the compact button clearly indicates its toggle function, maybe via a tooltip.
- **Shortcut Overlay**: `ShortcutOverlay` (`ui/components/shortcut_overlay.py`) uses a fade/show mechanism. Adding a CSS-like slide-in animation or a short `after` loop to visually slide the panel in would make the interaction feel significantly more premium ("Malcolm's Infusion").
- **Omnibar Suggestions**: The `Omnibar` has tooltip logic, but standard tooltips on Windows Tkinter/CustomTkinter can look out of place if not styled aggressively. Consider implementing a custom floating `ctk.CTkFrame` for tooltips that matches the app's dark theme perfectly.
- **Priority Grid Edit Mode**: The `_sync_edit_bar_state` shows/hides controls abruptly. A smooth expansion or crossfade between modes would improve the feel. Also, flashing the entry red on invalid input (`_flash_move_entry`) is functional but could include a subtle shake animation.
- **Empty States**: If stats scraping fails completely (no baseline, no network), the app should gracefully degrade and show a friendly "Offline Mode" indicator rather than blank data.

## 5. Other Suggestions & Polish

- **Typing & Docstrings**: The codebase is lightly typed. Adding standard Python type hints (`from typing import Optional, List, Dict, Tuple`) to method signatures (especially in `services/`) would make maintenance and IDE auto-completion much better.
- **Exception Handling**: There are several bare `except Exception as e:` and `pass` blocks (e.g., in `docking_loop` and `_tick`). These should at least `Logger.debug` the exception to help track down silent failures in production.
- **Testing**: Add a `tests/` directory with `pytest`. Core logic like priority queue sorting, asset path resolution, and stats scraping regex fallbacks are prime candidates for unit testing.
- **Constants**: Extract magic numbers (e.g., port 1700 for arena, hardcoded delays in `automation.py`) into a `constants.py` file to make tweaking behavior easier.
